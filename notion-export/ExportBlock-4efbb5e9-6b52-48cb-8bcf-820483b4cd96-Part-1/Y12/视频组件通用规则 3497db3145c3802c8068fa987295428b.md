# 视频组件通用规则

一、需求背景

当前平台 Web 双端、iOS-H5、App 端的 20 秒视频预览进度栏存在计时方式不统一问题（部分正计时、部分倒计时）。为规范三端交互逻辑，特统一 20 秒预览视频进度栏的计时规则。

二、**核心目标**

1.统一三端20秒视频预览进度栏的计时展示逻辑

2.保证进度栏交互逻辑三端一致

三、核心规则

1.计时方式统一要求

| 维度 | 具体规则 |
| --- | --- |
| 计时模式 | **强制采用倒计时模式，禁止使用正计时** |
| 显示格式 | 显示区：仅展示「剩余时长」，格式为 XX:XX（例：初始 00:20 → 播放中 00:15 → 结束 00:00）； |
| 进度条样式 | 1.填充方向：**从左至右**随播放进度填充；
2.进度比例：进度条进度（滑块位置）与剩余时长实时对应（如剩余10秒时进度栏填充50%，剩余5秒时填充75%，以此类推） |

2.交互逻辑统一要求

| 操作场景 | 交互规则 |
| --- | --- |
| 预览视频播放 | 1.自动播放时，倒计时从00:20 实时递减至00:00
2.暂停播放时，倒计时冻结在当前剩余时长
3.播放结束后，倒计时固定显示00:00，进度条完全填充，**禁止自动重播**
4 预览结束即刻弹出充值引导弹窗
5.关闭弹窗后，页面暂停，用户可自行点击播放按钮进行重复预览，预览结束后走3.4.5逻辑 |
| 进度条拖拽 | 预览视频进度条**禁止**拖拽，进度条仅展示播放进度，无交互 |
| 重复播放操作 | 点击「播放」按钮后，倒计时重置为00:00，进度条清空并重新开始填充 |

3.边界问题补充

| 场景 | 规则 |
| --- | --- |
| 网络卡顿/加载 | 视频加载中，倒计时显示00:20，进度条无填充；加载完成自动播放后，倒计时开始递减 |
| 未登录用户观看预览视频，结束后登录 | 1.登录后立即校验用户身份及视频计费模式；
2.若用户具备完整视频观看权限（如已解锁/会员/等级满足）；
立即播放完整视频；
进度栏从倒计时模式切换为正计时模式；
正计时初始显示
web：完整视频总时长
安卓/ios：00:00 /完整视频总时长（例：00:00/06:59）
进度条清空并重新从00:00开始填充；
3. 若用户无完整视频观看权限：继续播放预览视频，倒计时和进度条保持原有逻辑 |
| 已登录用户解锁视频 | 立即播放完整视频；
进度栏从倒计时模式切换为正计时模式；
正计时初始显示
web：完整视频总时长
安卓/ios：00:00 /完整视频总时长（例：00:00/06:59）
进度条清空并重新从00:00开始填充； |
| 预览中点击引导付费按钮 | **1.Web端：**
点击后立即弹出付费弹窗；
弹窗弹出后，预览视频持续播放，直至倒计时至00:00结束；
视频结束后按通用规则弹出充值引导弹窗；
关闭付费弹窗后，视频仍按原进度播放（未结束则继续，已结束则停留在00:00）。
**2.安卓/IOS-H5**
点击后立即弹出付费弹窗；
弹窗弹出后，预览视频持续播放，直至倒计时至00:00结束；
若用户点击弹窗内按钮跳转至会员充值页/金币充值页，返回预览页面时，视频为暂停状态
若用户仅关闭付费弹窗，视频仍按原进度播放至结束
视频结束后按通用规则弹出充值引导弹窗 |

### 2. 三端视频预览结束后充值引导弹窗交互规则需求文档

一、需求目的

二、核心目标

1.统一三端预览结束后的视频播放状态：禁止自动重播

2.统一三端充值引导弹窗的触发与交互逻辑

3.通过弹窗引导用户充值

三、核心规则

1.预览结束后视频播放规则（三端统一）

| 操作场景 | 规则 |
| --- | --- |
| 预览播放结束 | 1.20 秒预览视频播放至倒计时为00:00时，视频立即停止播放
2.**禁止视频自动循环预览**
3.进度栏保持完全填充状态，倒计时固定显示00:00
4.**预览视频停止播放的瞬间（倒计时为00:00时），弹窗立即弹出**
5.用户需手动点击播放按钮，再次预览 |

2.弹窗关闭后操作规则（三端统一）

| 操作场景 | 规则 |
| --- | --- |
| 关闭弹窗后 | 1.视频仍保持停止状态，需用户手动点击播放按钮才能重新播放预览视频；
2.点击播放按钮后，倒计时重置为00:20，进度栏清空，视频重新开始 20秒预览播放；
3.重播后再次播放至结束，仍触发上述弹窗规则。 |
